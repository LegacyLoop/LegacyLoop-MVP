File: app/api/shipping/rates/route.ts

Currently the dedup (line 46-56) only handles FedEx vs Shippo
duplicates. But Shippo itself returns multiple entries for
the same carrier+service (different account types or zones).

UPGRADE the dedup to also handle Shippo-internal duplicates:

  // Dedup — keep cheapest rate per carrier+service combo
  const seen = new Map<string, any>();
  for (const r of allRates) {
    const key = `${(r.provider || r.carrier || "").toLowerCase()}_${(r.servicelevel_name || r.service || "").toLowerCase()}`;
    const existing = seen.get(key);
    if (!existing) {
      seen.set(key, r);
    } else {
      // Keep the one with better source priority OR cheaper price
      const existingPrice = parseFloat(existing.amount || existing.rate || "0");
      const newPrice = parseFloat(r.amount || r.rate || "0");
      if (r.source === "fedex" && existing.source !== "fedex") {
        // Prefer FedEx direct
        seen.set(key, r);
      } else if (r.source === existing.source && newPrice < existingPrice && newPrice > 0) {
        // Same source — keep cheaper
        seen.set(key, r);
      }
    }
  }

This eliminates duplicate "UPS Ground" entries from Shippo
AND keeps the cheapest one when there are duplicates.

  — "Auth SUCCESS" or "Auth FAILED — status: XXX, body: YYY"

STEP 2 — If auth FAILS:
  Common FedEx sandbox issues:
  a) The API key format might need adjustment. FedEx keys
     sometimes need to be URL-encoded or have spaces trimmed.
  b) The grant_type might need to be "client_credentials"
     (verify this is correct for FedEx)
  c) The sandbox may require a specific account number.
     Try adding: grant_type: "client_credentials" AND
     scope: "CXS" (Customer Experience Sandbox)

  FedEx sandbox auth endpoint:
  POST https://apis-sandbox.fedex.com/oauth/token
  Content-Type: application/x-www-form-urlencoded
  Body: grant_type=client_credentials&client_id=KEY&client_secret=SECRET

  If this returns 401 or 403, the credentials may not be
  activated for the Rate API. Ryan may need to:
  — Log into FedEx Developer Portal
  — Ensure the Rate API is enabled for this project
  — Check if the sandbox credentials are active

STEP 3 — If auth SUCCEEDS but rates fail:
  The rate request body might not match FedEx's expected format.
  Try the SIMPLEST possible request:

  {
    "accountNumber": { "value": "" },
    "requestedShipment": {
      "shipper": { "address": { "postalCode": "04901", "countryCode": "US" } },
      "recipient": { "address": { "postalCode": "78704", "countryCode": "US" } },
      "pickupType": "DROPOFF_AT_FEDEX_LOCATION",
      "rateRequestType": ["LIST"],
      "requestedPackageLineItems": [{
        "weight": { "units": "LB", "value": 5 },
        "dimensions": { "length": 12, "width": 10, "height": 8, "units": "IN" }
      }]
    }
  }

  If this still fails, try removing accountNumber entirely.
  If it works, gradually add back fields.
