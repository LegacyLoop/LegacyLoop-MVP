ROOT CAUSE (verified by reading the code):
In app/api/shipping/center/route.ts line 55:
  aiShipping ? JSON.stringify(aiShipping) : null
This passes a JSON string like:
  {"weight":4.5,"length":14,"width":9,"height":0.7}

But suggestPackage() parameter #2 expects a
DIMENSION STRING like "14 x 9 x 0.7" — the
parseDimensions() regex at line 94-98 of
package-suggestions.ts only matches patterns
like "14 x 9 x 0.7" or "14x9x0.7".

The JSON format NEVER matches the regex, so
parseDimensions() returns null, and the
function falls back to CATEGORY-BASED sizing
(line 134-142) which gives "laptop" → Medium.

FIX: Convert the aiShipping object to a
proper dimension string before passing:
  const dimString = aiShipping
    ? `${aiShipping.length || ''} x ${aiShipping.width || ''} x ${aiShipping.height || ''}`
    : null;
