OBJECTIVE — FEDEX PAYMENT BLOCK FIX + PICKUP API OPTIMIZATION

================================================================================

This command does exactly TWO surgical fixes. No overhauls. No facelifts.

FIX 1 — FEDEX shippingChargesPayment (fixes ACCOUNT.NUMBER.MISMATCH)

Problem: FedEx OAuth2 auth SUCCEEDS (token received, 3599s expiry). Rate
requests return 400 Bad Request with error ACCOUNT.NUMBER.MISMATCH:
"As the payment Type is SENDER, ShippingChargesPayment Payor AccountNumber
should match the shipper account number."

Root cause: The FedEx Rate API requires a shippingChargesPayment block inside
requestedShipment when using an account number. Without it, FedEx defaults to
paymentType SENDER but cannot match the payor because we never specified who
is paying. The accountNumber at the top level is for API auth — the
shippingChargesPayment block is for billing/payment routing.

Fix: Add shippingChargesPayment block to both fedex-parcel.ts and fedex-ltl.ts.

Files: lib/shipping/fedex-parcel.ts, lib/shipping/fedex-ltl.ts

FIX 2 — PICKUP API CALL OPTIMIZATION (stops 56+ repeated calls)

Problem: Shipping Center fires /api/shipping/pickup/{itemId} for every
pickup-eligible item in TWO separate useEffect hooks:
  1. LocalPickupTab component (~line 4116) — fires on mount
  2. Parent component (~line 4528) — fires on every data change
With 14 items this produces 28+ calls minimum, multiplied by re-renders to 56+.

Fix: Add useRef guards to both useEffect hooks so each fires only once.

File: app/shipping/ShippingCenterClient.tsx
