File: app/api/shipping/center/route.ts

Currently metro estimates come from getMetroEstimates()
which uses a simplified formula. The results look the
same (~$14) for every city regardless of item weight.

Read lib/shipping/metro-estimates.ts to understand the
formula. The estimateShippingCost function takes fromZip
prefix, toZip prefix, and weight. But the results are
nearly flat because the formula doesn't differentiate
enough by weight and distance.

FIX: After calling getMetroEstimates(), POST-PROCESS
the results to add more weight-based variation:

  const metros = getMetroEstimates(
    item.saleZip || "04101",
    actualWeight
  );

  // Add realistic weight-based variation
  const adjustedMetros = metros.map(m => ({
    ...m,
    estimatedCost: Math.round(
      (m.estimatedCost + (actualWeight * 0.5) + (m.estimatedDays * 1.5))
      * 100
    ) / 100,
  }));

This adds $0.50/lb + $1.50/transit-day to each estimate,
making heavier items and farther cities cost more.
A 5 lb item to NYC (3 days) = base + $2.50 + $4.50 = ~$21
A 45 lb amp to LA (5 days) = base + $22.50 + $7.50 = ~$44
MUCH more realistic.
