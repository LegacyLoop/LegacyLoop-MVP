=== THE DATA BUG (Task 4 — most critical fix in this command) ===

The Shipping Center shows WRONG weights for items:
  - Fender Princeton Chorus Amp (real ~45 lbs) → shows "3 lbs"
  - Uzi Distortion Pedal (real ~2 lbs) → shows "12 lbs"

ROOT CAUSE traced through 3 files:

  1. app/api/shipping/center/route.ts, LINE 205:
     aiEstWeight: pkgSuggestion?.weightEstimate ?? aiShipping?.weight ?? null

     pkgSuggestion.weightEstimate comes from suggestPackage() which uses
     CATEGORY DEFAULTS in lib/shipping/package-suggestions.ts:
       "electronics" = 3 lbs  ← why the amp shows 3
       "instrument" = 12 lbs  ← why the pedal shows 12

     The REAL AI weight is in (item as any).aiWeightLbs (Prisma field,
     populated by populateFromAnalysis()). It's returned at line 191
     but aiEstWeight ignores it.

  2. app/shipping/ShippingCenterClient.tsx, weightStr() lines 41-45:
     Only checks item.weight (user-entered) then item.aiShipping?.weight.
     NEVER checks item.aiWeightLbs (the accurate AI analysis weight).

  3. app/shipping/ShippingCenterClient.tsx, getShippingMethod() line 171:
     Uses item.aiEstWeight (category default!) as primary weight source.
     Also line 172: reads aiBoxDims with ARRAY indices [0],[1],[2] but
     the API returns it as an OBJECT { length, width, height }. This
     silently returns 0 for all dims, making the >48" freight check
     never trigger from dimensions.

FIELD PRIORITY RULE (apply everywhere):
  User-entered weight > AI analysis weight > category default weight
  (item.weight) > (item.aiWeightLbs) > (item.aiEstWeight)
